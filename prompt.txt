## 角色与背景
我是一名中国高职院校教师，正为马来西亚留学生授课。我的母语是中文，学生的母语是英文。
课程项目是**AI芯片质量检测**，基于 YOLO（ultralytics）训练的目标检测模型。前面的同事已完成模型训练阶段，我负责的是模型应用环节。

## PPT 制作需求

请按以下结构制作 PPT，共分为三个部分：

### 第一部分：前期工作总结（模型训练回顾）
简要回顾同事已完成的工作，包含以下关键流程：
- 数据采集 → 数据清洗 → 数据标注 → 模型训练 → 模型测试
- 说明芯片检测模型的两类识别目标：good（好芯片）和 bad（坏芯片）

### 第二部分：模型效果分析
分析同学们自己训练的模型效果不理想的原因，引导学生理解深度学习的影响因素：
- 训练图片数据量不足
- 模型训练轮次（epochs）偏少
- 使用 CPU 训练耗时过长，影响迭代效率
- 可补充：数据标注质量、图片拍摄角度/光照对检测效果的影响

### 第三部分：应用案例实战——芯片质量检测
通过实际代码演示，介绍如何将训练好的模型应用到芯片检测中：

**1. Python 语言简介（1-2页）**
- Python 的特点：易学易读、丰富的第三方库、广泛应用在 AI 领域
- 与本次课程相关的库：ultralytics（YOLO）、opencv-python（图像处理）

**2. PyCharm 开发工具简介（1页）**
- PyCharm 的界面布局与常用功能
- 如何创建项目、安装依赖、运行 Python 脚本

**3. 单张图片检测代码讲解（detect_test_1.py）**
逐段讲解代码逻辑：
- 模型加载：`YOLO("chip.pt")`
- 图片读取：`cv2.imread()`
- 执行检测：`model(image_path)`
- 遍历检测结果：获取坐标 `box.xyxy`、类别 `box.cls`
- 颜色区分：好芯片蓝色框 `(255,0,0)`、坏芯片红色框 `(0,0,255)`
- 数量统计与结果显示：`cv2.putText()` 在图片左上角显示 Good/Bad 计数
- 保存结果图片与弹窗展示

**4. 批量图片检测代码讲解（detect_test_2.py）**
- 遍历 `datasets/` 文件夹中的所有图片
- 每张图片独立检测并统计
- 所有图片处理完毕后输出汇总报告（总图片数、总好芯片数、总坏芯片数）

## PPT 格式要求
- **双语呈现**：所有文字内容需同时包含**中文和英文**，英文在前或中文在前均可，但每页保持统一。建议关键术语保留英文原词（如 YOLO、epochs、bounding box）
- **风格一致**：参考之前课程使用的 PPT 模板风格，保持配色与排版统一
- **图文并茂**：代码讲解页面建议左侧放代码截图，右侧标注关键行说明
- **适量代码**：每个代码讲解页面控制在 1-2 个核心代码片段，不要整页堆满代码
